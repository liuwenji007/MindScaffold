# 阿窝睡了存储迁移方案（无感升级）

## 迁移目标

- 让新版本统一使用新品牌 key：
  - 卡片 key：`awoshuile_cards_*`
  - 设置 key：`awo_*`
- 兼容旧版本 key：
  - 卡片 key：`mindscaffold_cards_*`
  - 设置 key：`ms_*`
- 保证用户升级后历史记录不丢失、设置项不中断。

## 迁移策略

## 1) 读路径兼容（优先新 key，回退旧 key）

- 读取单条卡片：先读 `awoshuile_cards_*`，未命中再读 `mindscaffold_cards_*`。
- 读取设置项：先读 `awo_*`，未命中再读 `ms_*`。

## 2) 迁移触发（命中旧 key 即迁移）

- 单条读取命中旧卡片后，立即写入新 key 并删除旧 key。
- 读取设置项命中旧 key 后，立即写入新前缀并删除旧前缀。
- 应用启动时执行一次全量迁移扫描，清理遗留旧卡片 key。

## 3) 写路径统一（仅写新 key）

- 新增/更新卡片仅写入 `awoshuile_cards_*`，并尝试删除同 id 的旧 key。
- 设置项仅写入 `awo_*`，并清理 `ms_*`。

## 4) 删除与清空双前缀处理

- 删除单条卡片时同时删除新旧前缀 key。
- 清空数据时批量清理新旧两套卡片 key。

## 验收用例

- 旧版本已有卡片数据，升级后历史页仍可展示。
- 旧版本设置项可在新版本读取到，并完成自动迁移。
- 迁移后重复进入应用，不会出现重复卡片。
- 执行“清空所有记录”后，新旧卡片 key 均被清理。
