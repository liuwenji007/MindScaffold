# Todolist 模板与规范

## 组织方式

根据需求规模选择组织方式：

### 按功能模块（推荐）

```markdown
## 模块A：用户管理

### 基础 CRUD
- [ ] [P0] 实现用户列表页（Table + 分页 + 搜索）
- [ ] [P0] 实现新建用户弹窗（表单校验 + 提交）
- [ ] [P0] 实现编辑用户（复用新建表单，预填数据）
- [ ] [P1] 实现删除用户（确认弹窗 + 批量删除）

### 高级功能
- [ ] [P1] 实现用户角色分配（多选 + 树形结构）
- [ ] [P2] 实现用户导入导出（Excel 模板 + 进度反馈）

## 模块B：权限管理
- [ ] ...
```

### 按优先级

```markdown
## P0 — 核心功能（本迭代必须完成）
- [ ] 实现xx功能
- [ ] 实现xx功能

## P1 — 重要功能（本迭代尽量完成）
- [ ] 实现xx功能

## P2 — 优化项（下一迭代）
- [ ] 优化xx体验
```

### 按开发阶段

```markdown
## 阶段一：基础框架（第 1-2 天）
- [ ] 搭建页面路由结构
- [ ] 定义 TypeScript 类型

## 阶段二：核心功能（第 3-5 天）
- [ ] 实现主要 CRUD
- [ ] 对接后端 API

## 阶段三：完善体验（第 6-7 天）
- [ ] 添加 loading/empty 状态
- [ ] 错误处理与提示
```

## 任务写法规范

### 格式

```
- [ ] [优先级] 动作动词 + 对象 + 可选验收标准
  - 依赖：依赖的其他任务
  - 备注：技术细节或注意事项
```

### 动作动词

| 动词 | 含义 | 示例 |
|------|------|------|
| 实现 | 从零开发 | 实现用户列表页 |
| 添加 | 在已有基础上增加 | 添加搜索筛选功能 |
| 修改 | 改动已有逻辑 | 修改状态流转规则 |
| 优化 | 改善性能/体验 | 优化列表加载性能 |
| 重构 | 结构性调整 | 重构表单校验逻辑 |
| 对接 | 连接外部 | 对接用户列表 API |
| 配置 | 设置/调整配置 | 配置路由权限守卫 |

### 粒度控制

- **太粗**：❌ 实现用户管理模块
- **太细**：❌ 给按钮添加 onClick 事件
- **合适**：✅ 实现用户列表页（Table + 分页 + 搜索筛选）

原则：每个任务 **0.5~2 天** 的工作量，可独立验证完成。

### 依赖标注

```markdown
- [ ] [P0] 定义 User/Role TypeScript 类型
- [ ] [P0] 实现用户列表 API 封装
  - 依赖：TypeScript 类型定义
- [ ] [P0] 实现用户列表页 UI
  - 依赖：API 封装、TypeScript 类型
```
