# 需求输入来源处理

## PRD 文档

### Markdown / 文本文件
- 直接读取文件内容
- 识别标题层级结构
- 提取需求描述、功能清单、原型说明
- 注意删除线内容（`~~文字~~`）视为已废弃

### 在线文档链接
- 提示用户将内容粘贴到对话中
- 或者导出为 Markdown/PDF 后作为文件输入

### 图片/设计稿
- 识别页面布局和交互元素
- 提取标注的文字、按钮、表单等
- 判断是 B 端（管理后台）还是 C 端（用户端）页面
- 注意区分当前页面属于哪个模块

## 口头描述

用户直接描述需求时：

1. **提炼核心功能**：从描述中识别主要功能点
2. **追问关键细节**：
   - 用户角色有哪些？
   - 核心操作流程是什么？
   - 有没有现成的设计稿或 PRD？
   - 是新功能还是在现有模块上修改？
3. **确认范围**：明确哪些在本次开发范围内
4. **结构化整理**：将口头信息按五维度框架整理

## 已有代码上下文

当在现有项目上开发时：

1. 用 `analyze_project` 工具分析项目结构
2. 定位需求涉及的模块和文件
3. 理解现有的数据模型和 API 结构
4. 评估改动影响范围

## 输入预处理

无论来源，预处理步骤：

1. **去噪**：去除格式标记、页眉页脚、无关内容
2. **结构化**：将输入按功能模块初步分组
3. **标记不确定项**：标记需要确认的模糊需求
4. **提取关键词**：识别技术术语、业务概念、模块名称
