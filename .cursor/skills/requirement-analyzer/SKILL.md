# Requirement Analyzer

需求分析与结构化专家，帮助将业务需求转化为结构化的分析文档和可执行的 todolist。

## 工作流程

需求分析遵循以下步骤：

1. **获取需求输入** - 从飞书 PRD、手动输入或文件读取需求
2. **需求结构化分析** - 按分析框架进行深入分析
3. **生成分析文档** - 输出结构化的需求分析文档
4. **生成 Todolist** - 基于分析结果生成可执行的开发任务列表

## 快速开始

### 1. 获取需求输入

根据需求来源选择相应的处理方式：

**飞书 PRD 文档：**
- 使用 Feishu MCP 读取文档（通过 token 或链接）
- 解析文档结构，提取功能需求、业务规则、交互要求
- 展示解析后的需求摘要，请用户确认

**手动输入：**
- 引导用户结构化描述需求（功能名称、功能点、交互流程、数据需求、特殊要求）
- 进行需求澄清和补充
- 整理成结构化需求文档

**文件输入：**
- 读取需求文档文件（支持 .md, .txt, .docx）
- 解析文档内容，提取关键信息
- 展示解析结果，请用户确认

详细处理指南见 [references/input-sources.md](references/input-sources.md)。

### 2. 需求结构化分析

按照分析框架进行五个维度的分析：

1. **业务场景分析** - 识别核心业务目标、用户角色、功能边界
2. **功能点拆解** - 列出功能点、评估优先级、确定依赖关系
3. **交互流程梳理** - 绘制操作流程、识别交互节点、设计异常处理
4. **数据模型分析** - 识别数据结构、确定数据来源、分析数据流转
5. **技术实现评估** - 评估复杂度、识别技术难点、提出实现方案

详细分析框架见 [references/analysis-framework.md](references/analysis-framework.md)。

### 3. 生成分析文档

将分析结果整理成结构化的文档，包含：
- 需求概述
- 业务场景分析结果
- 功能点清单（含优先级）
- 交互流程图
- 数据模型设计
- 技术实现方案

### 4. 生成 Todolist

基于需求分析结果，生成可执行的开发任务列表：

- 按功能模块组织，或按优先级组织，或按开发阶段组织
- 每个任务包含：动作动词 + 具体对象 + 验收标准（可选）
- 标注任务依赖关系
- 根据功能复杂度选择合适的任务粒度

详细模板见 [references/todolist-template.md](references/todolist-template.md)。

## 分析原则

- **结构化**：使用统一的框架进行分析，确保不遗漏关键维度
- **完整性**：深入挖掘需求细节，识别边界情况和异常处理
- **可执行**：生成的分析文档和 todolist 应该清晰、具体、可执行
- **可追溯**：保持需求来源与分析结果的关联，便于后续追溯

## 输出格式

### 需求分析文档结构

```markdown
# 需求分析文档：[功能名称]

## 1. 需求概述
[功能描述、目标用户、业务价值]

## 2. 业务场景分析
[核心业务目标、用户角色、功能边界]

## 3. 功能点拆解
[功能清单、优先级、依赖关系]

## 4. 交互流程梳理
[操作流程、交互节点、异常处理]

## 5. 数据模型分析
[数据结构、数据来源、数据流转]

## 6. 技术实现评估
[复杂度评估、技术难点、实现方案]
```

### Todolist 格式

根据需求选择合适的组织方式（功能模块/优先级/开发阶段），参考 [references/todolist-template.md](references/todolist-template.md)。
