# Todolist 生成模板

## 结构化 Todolist 格式

基于需求分析结果，生成结构化的 todolist，包含以下维度：

### 1. 按功能模块组织

```markdown
## 功能模块：[模块名称]

### [功能点 1]
- [ ] 任务 1：具体描述
- [ ] 任务 2：具体描述
- [ ] 任务 3：具体描述

### [功能点 2]
- [ ] 任务 1：具体描述
- [ ] 任务 2：具体描述
```

### 2. 按优先级组织

```markdown
## P0 - 必须完成
- [ ] 任务描述（所属功能模块）

## P1 - 重要功能
- [ ] 任务描述（所属功能模块）

## P2 - 可选功能
- [ ] 任务描述（所属功能模块）
```

### 3. 按开发阶段组织

```markdown
## 阶段 1：基础功能
- [ ] 任务描述

## 阶段 2：核心功能
- [ ] 任务描述

## 阶段 3：增强功能
- [ ] 任务描述
```

## Todolist 任务描述规范

每个任务应该包含：
1. **动作动词**：创建、实现、添加、优化等
2. **具体对象**：组件名、功能名、接口名等
3. **验收标准**：如何判断任务完成（可选）

示例：
- ✅ 创建用户列表组件，支持分页和搜索
- ✅ 实现用户详情 API 调用，包含错误处理
- ✅ 添加表单验证，确保必填字段不为空

## 任务粒度建议

- **细粒度**：适合复杂功能，每个任务 1-4 小时
- **中粒度**：适合常规功能，每个任务 4-8 小时
- **粗粒度**：适合简单功能，每个任务 1-2 天

根据功能复杂度选择合适的粒度。

## 依赖关系标注

对于有依赖关系的任务，使用以下标注：
- `[依赖: 任务ID]` - 标注依赖的任务
- `[前置: 任务ID]` - 标注前置任务

示例：
- [ ] 实现用户列表组件 `[依赖: 用户列表 API]`
- [ ] 添加用户搜索功能 `[前置: 用户列表组件]`
