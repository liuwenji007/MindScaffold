# 业务状态码参考

## 状态码定义

本项目使用业务状态码来标识接口调用的结果，HTTP 状态码通常为 200，实际业务状态通过响应体中的 `code` 字段表示。

### 成功状态码

| 状态码 | 说明 | 使用场景 |
|--------|------|----------|
| S00000 | 成功 | 接口调用成功，业务处理正常 |

### 错误状态码

| 状态码 | 说明 | 使用场景 |
|--------|------|----------|
| E00001 | 系统错误 | 服务器内部错误、异常 |
| E00002 | 网络异常 | 网络连接问题、超时 |
| E00003 | 请求参数错误 | 请求参数格式错误、缺少必填参数 |
| E00004 | 验证失败 | 数据验证不通过、格式校验失败 |
| E00005 | 您还未登录 | 用户未登录或登录已过期 |
| E00006 | 数据已过期 | 数据已过期、需要刷新 |
| E00007 | 权限不足 | 用户没有操作权限 |
| E00008 | 次数超过限制 | 操作次数超过限制 |
| E00009 | 频次超限 | 请求频率超过限制 |
| E00010 | 资源或者数据不存在 | 查询的资源不存在 |
| E00011 | 资源或者数据已经存在 | 创建的资源已存在 |
| E00012 | 格式不合法 | 数据格式不符合要求 |
| E00013 | 请求被限制或者拒绝 | 请求被系统限制或拒绝 |
| E00014 | 数据格式错误 | 数据格式错误、无法解析 |
| E00015 | 功能已不支持 | 功能已废弃、不再支持 |
| E00016 | 数据或版本不匹配 | 数据版本不匹配、需要更新 |

## 响应结构

### 成功响应

```json
{
  "code": "S00000",
  "message": "成功",
  "data": {
    // 实际业务数据
  },
  "success": true
}
```

### 错误响应

```json
{
  "code": "E00003",
  "message": "请求参数错误",
  "data": {},
  "success": false
}
```

## 状态码分类

### 系统级错误（E00001-E00002）
- 服务器内部问题
- 网络连接问题
- 通常需要联系技术支持

### 客户端错误（E00003-E00005）
- 请求参数问题
- 验证失败
- 登录状态问题
- 前端可以提示用户修正

### 权限和限制错误（E00006-E00009）
- 数据过期
- 权限不足
- 操作限制
- 需要用户重新操作或联系管理员

### 数据错误（E00010-E00012）
- 资源不存在
- 资源已存在
- 格式不合法
- 前端可以提示用户或引导操作

### 业务错误（E00013-E00016）
- 请求被限制
- 数据格式错误
- 功能不支持
- 版本不匹配
- 需要根据具体错误提示用户

## 处理建议

### 前端处理

1. **成功处理**：`code === "S00000"` 时，正常处理业务数据
2. **错误处理**：根据错误码显示相应的错误提示
3. **登录处理**：`code === "E00005"` 时，跳转到登录页
4. **权限处理**：`code === "E00007"` 时，提示权限不足
5. **参数错误**：`code === "E00003"` 时，显示参数错误提示

### 接口文档整理

在整理接口信息时，应该：
1. 明确标注 HTTP 状态码（通常是 200）
2. 列出可能的业务状态码
3. 提供成功和错误响应示例
4. 说明各错误码的处理方式
